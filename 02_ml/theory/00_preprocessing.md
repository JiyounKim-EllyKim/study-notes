# Preprocessing 개념 정리

## 1. 데이터 클렌징
- 결측치/중복값/이상치 처리(대치/제거)
- 결측치가 많은 행이거나, 결측치가 있는 행이 많지 않은 경우 제거를 함.
- 결측치 대치 처리는 주로 평균/중앙값/최빈값 중 하나로 대치를 한다.

## 2. 데이터 인코딩
- Label Encoding: 범주형 변수마다 하나의 숫자를 부여하고, 해당 범주를 해당 숫자로 표시(알파벳 순으로, 숫자는 0부터 시작)
- One-hot Encoding: 범주형 변수마다 하나의 열을 부여하고, 해당 범주에만 1을 표시(OVR: one vs rest 또는 OVA: one vs all) 

## 3. 데이터 스케일링
- 표준화(Standardization): 평균을 0, 표준편차를 1로 변환하는 방식
    - 데이터가 정규분포를 따를 때 효과적
    - 음수 값도 허용
    - 이상치(outlier)의 영향을 덜 받음
- 정규화(Normalization): 데이터를 0과 1 사이로 스케일링
    - 특정 범위(보통 0~1)로 제한
    - 이상치에 민감
    - 거리 기반 알고리즘(k-NN, SVM 등)에 자주 사용
- 로버스트 스케일링(Robust Scaling): 중앙값(Median)과 사분위범위(IQR)을 이용한 스케일링
    - 이상치에 강건(Robust)
    - 중앙값과 IQR 기준 -> 데이터 분포 왜곡이 적음
    - StandardScaler보다 이상치 영향이 적은 대안
    - 다양한 분포의 데이터에 적용 가능
